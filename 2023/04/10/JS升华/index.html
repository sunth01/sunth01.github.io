    this 的指向

    • 以函数形式调用时，this 永远都是 window
    • 以方法的形式调用时，this 是调用方法的对象
    • 以构造函数的形式调用时，this 是新创建的那个对象
    • 使用 call 和 apply 调用时，this 是指定的那个对象
      • call 和 apply 的区别：两者的作用相同，都是将 this 绑定到某个对象上，并立即执行这个函数。它们的主要区别在于传递参数的方式不同：call() 方法接收一个或多个参数列表，apply() 方法接收一个数组参数
      • bind() 方法会返回一个新的函数，它的 this 值被绑定到了传入的值。不同于 call() 和 apply()，bind() 不会调用函数，而是返回修改前的函数，稍后需要调用才能执行
    • 箭头函数：箭头函数的 this 看外层是否有函数如果有，外层函数的 this 就是内部箭头函数的 this 如果没有，就是 window

    flex 布局

    **flex-grow** 属性定义项目的放大比例，默认为 0，即如果存在剩余空间，也不放大。如果所有项目的 flex-grow 属性都为 1，则它们将等分剩余空间

    **flex-shrink** 属性定义了项目的缩小比例，默认为 1，即如果空间不足，该项目将缩小。如果所有项目的 flex-shrink 属性都为 1，当空间不足时，都将等比例缩小。如果一个项目的 flex-shrink 属性为 0，其他项目都为 1，则空间不足时，前者不缩小。

    **flex-basis** 属性定义了在分配多余空间之前，项目占据的主轴空间。浏览器根据这个属性，计算主轴是否有多余空间。它的默认值为 auto，即项目的本来大小

    水平垂直居中

    • 父盒子开启 flex,并设置主轴居中,侧轴居中
    • 父盒子开启 flex,子盒子设置 margin:auto
    • 子绝父相,子盒子设置 top 和 left 值为 50%,利用 transform 基于自身回去-50%
    • 子绝父相,子盒子设置上下左右均为 0,在设置 margin:auto 也可以

    盒模型

    • 四部分: 内容+内边距+外边距+边框
    • 盒模型可以进行切换: box-sizing: border-box; ie 盒模型/c3 盒模型/怪异盒模型, 宽高定死,不受边框内间距撑开
    • 默认是 content-box: 标准盒模型, width+内间距+边框

    JS延迟加载的方式

    • defer：只适用于外部脚本文件，放在script标签中，脚本背延迟到整个页面都解析完毕之后，在执行script标签中的defer属性，等于告诉浏览器立即下载但延迟执行
    • async：不让页面等待脚本下载和执行，从而异步加载页面其他内容，也是存放在script标签中

    call,apply,bind的区别

    都是用来改变this指向的，第一个参数都是this要指向的对象

    • call方法接收的是若干个参数列表，apply接收的是一个包含多个参数的数组
    • bind方法创建一个新的函数，当这个函数调用时，其this值为其提供的值，

    EventLoop事件循环机制

    宏任务 - script，setTimeout，setInterval
    微任务 - Promise.then，promise.nextTick

    • 同步任务与异步任务执行，在主线程上先执行第一个同步任务，只有同步任务执行完毕才会执行后面的代码，执行过程中遇到异步任务时，将其放入任务队列，继续接着往下执行别的代码，执行完同步代码去看任务队列里面有没有要执行的异步代码，有的话取出来执行，任务队列根据异步类型又分为宏任务与微任务,宏任务放到宏任务队列,微任务放到微任务队列
